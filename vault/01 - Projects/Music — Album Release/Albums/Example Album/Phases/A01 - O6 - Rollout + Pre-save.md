# A01 O6 — Rollout + Pre-save

## Overview

Submit the album to the distributor (wizard left at review step — never auto-submitted). Set up the pre-save campaign. Schedule launch-day social posts. The release date should be ≥ 4 weeks out for editorial / pre-save eligibility on most platforms.

## Tasks

- [ ] Apply [[08 - System/Agent Directives/launch-ops.md|launch-ops]].
- [ ] Drive the distributor wizard via `music-distro` — fill in album metadata, upload mastered tracks, upload cover art, set release date.
- [ ] **Stop at the wizard's review step.** Operator clicks Confirm; the agent does NOT auto-submit (this costs money).
- [ ] Set up the pre-save campaign on the distributor's pre-save tool (Linkfire / Hypeddit / distributor-native — depends on distributor).
- [ ] Schedule launch-day social posts via `instagram-publish`, `tiktok-publish`, etc. Posts include cover art reveal, lyric snippet, pre-save link.
- [ ] Update the album's `## Publish checklist` rows.

## Acceptance Criteria

- [ ] Distributor wizard sits at review step with all fields filled in. **Operator confirm pending — DO NOT mark phase completed until operator confirms.**
- [ ] Pre-save campaign live with correct release date.
- [ ] Launch-day social posts queued (not yet posted).
