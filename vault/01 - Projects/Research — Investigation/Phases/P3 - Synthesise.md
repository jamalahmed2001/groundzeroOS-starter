# P3 — Synthesise

## Overview

Map the evidence. Where do the sources agree? Where do they disagree? What's strong, what's weak, what's missing? The output is a synthesis doc — the skeleton of the eventual deliverable, but not the deliverable itself.

## Tasks

- [ ] Read every source note from `Sources/`.
- [ ] For each sub-question, write a synthesis paragraph:
  - What does the evidence say, summarised?
  - Where do credible sources agree?
  - Where do they disagree, and what's the most likely cause of the disagreement (different populations, different methods, different eras, genuine open question)?
  - What's the evidence quality — is the agreement on strong or weak ground?
- [ ] List explicit gaps — sub-questions where the evidence doesn't yet support an answer.
- [ ] Identify the 2-3 *load-bearing* findings — claims the deliverable will rest on. These need the strongest sourcing.
- [ ] Write `Phases/P3 - Synthesis.md` with all of the above.

## Acceptance Criteria

- [ ] Synthesis paragraph per sub-question.
- [ ] Gaps named explicitly.
- [ ] Load-bearing findings identified and traceable to sources.
- [ ] No new source fetching (if a gap demands it, surface as blocker — re-run P2).
